1. Project Overview

The Gas Boiler Web Application is a full-stack monitoring and control system for gas boilers installed in buildings. It enables users to register buildings on an interactive map, attach gas boilers, monitor indoor/outdoor temperatures in real time, receive alarms for anomalies, and analyse historical heating data via charts.

🏢 Building Management

Create, edit, delete buildings on an interactive OpenStreetMap map. Specify thermal properties (U-values, surface areas, desired temperature).

🔥 Boiler Management

Attach one or more gas boilers per building. Configure max power and efficiency. Real-time power output via P-controller.

⚡ P-Controller (Real-Time)

Background proportional controller runs every minute, adjusts boiler power to maintain desired indoor temperature. Pushes live updates via SignalR.

🚨 Alarm System

Automatic alarms for: insufficient capacity, high/low indoor temp, high/low outdoor temp, high daily cost. Real-time notifications via SignalR.

📊 Charts & Analytics

Historical charts for temperature, heat loss, power requirements, and daily heating costs. Filter by building and date range.

📁 Data Management

Admin can configure recording interval, generate test data, export CSV, and clear all historical data.

👥 User Management

Role-based access (Admin / User). Admin can view all data, block/unblock users. Each user sees only their own buildings.

⚙️ System Parameters

Admin configures global default U-values, gas price, safety factor, boiler efficiency, thermal mass, temperature step — used in heat-loss calculations.

7.2 BuildingCalculatorService (Services/BuildingCalculatorService.cs)

Core heat-loss calculation engine. Formula: Q = U × A × ΔT

// Heat loss per element (Watts)
double wallLoss    = building.WallUValue    * building.WallArea    * tempDiff;
double windowLoss  = building.WindowUValue  * building.WindowArea  * tempDiff;
double ceilingLoss = building.CeilingUValue * building.CeilingArea * tempDiff;
double floorLoss   = building.FloorUValue   * building.FloorArea   * tempDiffGround; // uses ground temp

// Apply safety factor
double totalHeatLoss = (wallLoss + windowLoss + ceilingLoss + floorLoss) * safetyFactor;
double requiredPowerKw = totalHeatLoss / 1000.0;

// Cost calculation
double dailyEnergyKwh = (totalHeatLoss * 24) / 1000.0;
double dailyCost  = (dailyEnergyKwh / avgEfficiency) * gasPricePerKwh;
double monthlyCost = dailyCost * 30;
double annualCost  = dailyCost * 365;

Uses OpenWeatherMap API for live outdoor temp, falls back to design temp if API fails.

8. Background Services (Hosted)

8.1 BoilerControlBackgroundService (Services/BoilerControlBackgroundService.cs)

Runs every 1 minute. Implements a Proportional (P) controller to regulate indoor temperature.

// Constants
private const double Kp = 1.5;       // proportional gain
private const double Deadband = 0.3; // ±0.3°C tolerance

// Per building:
var error = building.DesiredTemperature - indoorTemp;
var feedForward = reading.HeatLossWatts / 1000.0; // kW

if (indoorTemp > building.DesiredTemperature)
    newTotalPower = 0;                      // above target → boilers OFF
else if (Math.Abs(error) <= Deadband)
    newTotalPower = feedForward;            // within deadband → maintain heat loss
else
    newTotalPower = Math.Clamp(feedForward + error * Kp, 0, totalAvailablePower);

// Distribute equally across boilers
var powerPerBoiler = newTotalPower / boilers.Count;
foreach (var boiler in boilers)
    boiler.CurrentPower = Math.Clamp(powerPerBoiler, 0, boiler.MaxPower);

Indoor Temperature Calculation

Physics-based model using building volume and thermal mass:

var heatGenerated = boilers.Sum(b => b.CurrentPower * b.Efficiency * 1000.0); // W
var heatLost = reading.HeatLossWatts;
var netHeat = heatGenerated - heatLost;

var energyChange = netHeat * timeStep;                    // Joules
var deltaT = energyChange / (thermalMass * volume);       // °C change
var newIndoor = previousIndoor + deltaT;

// Realistic clamp
newIndoor = Math.Clamp(newIndoor,
    reading.OutdoorTemperature - 10,
    building.DesiredTemperature + 15);

After calculation, broadcasts via SignalR:

await _hubContext.Clients.All.SendAsync("IndoorTemperatureUpdated", new {
    buildingId, buildingName, temperature, desiredTemperature,
    error, outdoorTemperature, hasBoilers, timestamp

8.2 DataRecordingBackgroundService (Services/DataRecordingBackgroundService.cs)

Runs at a configurable interval (stored in DataManagementSettings, default 60 min). Polls for interval changes every 30 seconds — no restart needed after changing the interval.

while (!stoppingToken.IsCancellationRequested)
{
    int intervalMinutes = await GetRecordingIntervalAsync(); // reads DB
    await RecordDataAsync();  // calls HistoricalDataService + AlarmService.CheckAndCreateAlarmsAsync()

    // Wait with interval-change detection
    var elapsed = 0;
    while (elapsed < intervalMinutes * 60 && !stoppingToken.IsCancellationRequested)
    {
        await Task.Delay(TimeSpan.FromSeconds(30), stoppingToken);
        elapsed += 30;
        var newInterval = await GetRecordingIntervalAsync();
        if (newInterval != intervalMinutes) break; // interval changed — restart cycle
    }
}

10. Authentication — JWT Helper (Helpers/JwtHelper.cs)

public string GenerateJwtToken(User user)
{
    var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(secretKey!));
    var credentials = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);

    var claims = new[]
    {
        new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()),
        new Claim(ClaimTypes.Name,  user.Username),
        new Claim(ClaimTypes.Email, user.Email),
        new Claim(ClaimTypes.Role,  user.Role)       // "Admin" | "User"
    };

    var token = new JwtSecurityToken(
        issuer:   jwtSettings["Issuer"],
        audience: jwtSettings["Audience"],
        claims:   claims,
        expires:  DateTime.UtcNow.AddMinutes(ExpirationMinutes),
        signingCredentials: credentials);

    return new JwtSecurityTokenHandler().WriteToken(token);
}

14. Frontend — SignalR Hooks

14.1 useSignalR (src/hooks/useSignalR.ts)

Manages the SignalR HubConnection lifecycle. Exponential backoff reconnect, max 30 s delay.

export const useSignalR = (hubUrl: string, token: string): SignalRConnection => {
  const connection = new signalR.HubConnectionBuilder()
    .withUrl(hubUrl, { accessTokenFactory: () => token })
    .withAutomaticReconnect({
      nextRetryDelayInMilliseconds: (ctx) =>
        Math.min(2000 * Math.pow(2, ctx.previousRetryCount), 30000)
    })
    .build();

  // start connection, expose { connection, isConnected, error }
};

14.2 useSignalREvent (src/hooks/useSignalR.ts)

Registers/deregisters a typed event handler on a hub connection:

export const useSignalREvent = <T>(
  connection: HubConnection | null,
  eventName: string,
  handler: (data: T) => void
) => {
  useEffect(() => {
    if (!connection) return;
    connection.on(eventName, handler);
    return () => connection.off(eventName, handler);
  }, [connection, eventName, handler]);
};

14.3 useRealtimeBoilers (src/hooks/useRealtimeBoilers.ts)

Subscribes to IndoorTemperatureUpdated, BoilerPowerUpdated, and CapacityWarning events. Returns Maps keyed by buildingId/boilerId.

export const useRealtimeBoilers = (token: string) => {
  const { connection, isConnected } = useSignalR("https://localhost:44314/boilerHub", token);

  useSignalREvent<IndoorTempUpdate>(connection, "IndoorTemperatureUpdated", (data) => {
    setTemperatures(prev => { const m = new Map(prev); m.set(data.buildingId, data); return m; });
    setLastUpdate(new Date());
  });

  // Also handles BoilerPowerUpdated, CapacityWarning
  return { boilerPowers, temperatures, capacityWarnings, lastUpdate, isConnected };
};
